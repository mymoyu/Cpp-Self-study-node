Mysql逻辑架构四层: 连接层(应用API调用), 服务层(数据库逻辑设计), 引擎层(可拔插引擎), 存储层(硬件)

##### 存储引擎

show engines; 查看有的引擎

|  对比项  |                          MyISAM                          |                            InnoDB                            |
| :------: | :------------------------------------------------------: | :----------------------------------------------------------: |
|  主外锁  |                          不支持                          |                             支持                             |
|   事务   |                          不支持                          |                             支持                             |
|  行表锁  | 表锁, 即使操作一条记录也会锁住整个表, 不适合高并发的操作 |   行锁, 操作时只锁某一行, 不对其他行有影响, 适合高并发操作   |
|   缓存   |                只缓存索引, 不缓存真实数据                | 不仅缓存索引还有缓存真实数据, 对内存要求较高, 而且内存大小对性能有决定性的影响 |
|  表空间  |                            小                            |                              大                              |
|  关注点  |                           性能                           |                             事务                             |
| 默认安装 |                            Y                             |                              Y                               |
